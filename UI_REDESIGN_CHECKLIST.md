# MoonBTI UI 리디자인 체크리스트 (실무 실행안)

## 목표
- 완주율/정답 입력 정확도 개선
- 가독성 및 접근성(텍스트/터치/키보드/스크린리더) 강화
- 뉴모피즘 아이덴티티는 유지하되 정보 위계 명확화

---

## 0) 현재 문제 정의 (핵심)
- 텍스트 크기/자간이 작고 과해 가독성이 낮음
- 카드/그림자 밀도가 높아 핵심 CTA 대비가 약함
- 선택 즉시 다음 문항으로 넘어가 통제감이 낮음
- `GestureDetector` 중심 상호작용으로 접근성 대응이 약함

---

## 1) 우선순위 P0 (이번 스프린트 필수)

### P0-1. 타이포 스케일 통일
- [x] 공통 텍스트 스케일 정의
  - Display: 40
  - H1: 28
  - H2: 22
  - Body: 16
  - BodySmall: 14
  - Caption: 12
- [x] 한글 자간 규칙 통일
  - 본문/설명: 0 ~ 0.2
  - 라벨/배지: 0.3 ~ 0.8 (2.0 이상 금지)
- [x] 본문 line-height 1.45 ~ 1.7로 통일

**완료 기준 (DoD)**
- 홈/문항/결과에서 12 미만 폰트가 없음
- 본문은 최소 14 이상
- title/body/caption 시각 위계가 첫 화면 3초 내 인지 가능

### P0-2. 문항 플로우 통제감 개선
- [x] 답변 선택 후 즉시 이동 제거
- [x] 선택 상태 표시 + `다음` 버튼 활성화 방식으로 변경
- [x] `이전` 이동 시 기존 선택값 유지
- [x] 마지막 문항에서 `결과 보기` 버튼으로 전환

**완료 기준 (DoD)**
- 오답 클릭 후 1회 탭으로 수정 가능
- 사용자가 명시적으로 다음 단계로 이동
- 문항 완료까지 이탈 없는 플로우 확인

### P0-3. 버튼/선택 컨트롤 접근성 강화
- [x] `GestureDetector` 단독 버튼을 `Semantics + Material + InkWell` 기반으로 전환
- [x] 최소 터치 영역 44x44 보장
- [x] 포커스/호버/프레스 상태 시각 피드백 제공
- [x] VoiceOver 라벨(예: "매우 그렇다, 5점") 제공

**완료 기준 (DoD)**
- 키보드(Tab/Enter)로 주요 액션 수행 가능
- 스크린리더로 버튼 역할/라벨/선택상태 인식 가능

### P0-4. 정보 위계 재정렬
- [x] 화면당 강한 그림자 레벨 최대 2개
- [x] 장식용 NeuBox 축소, 핵심 카드/CTA만 강조
- [x] 상단/본문/하단 액션의 시선 흐름 고정

**완료 기준 (DoD)**
- 홈 화면에서 시선 순서가 `로고 → 타이틀 → 설명 → 시작`으로 자연스럽게 흐름
- 결과 화면에서 `유형 카드 → 요약 → 상세` 순서 인지 가능

## 6) 팀 운영 팁 (실무)
- 한 번에 전부 바꾸지 말고 P0만 먼저 머지
- 사용자 테스트는 5명만 해도 패턴이 보임
- 지표는 최소 2개 추적: `완주율`, `문항당 체류시간`
- 디자인 리뷰 기준은 "예쁨"보다 "읽힘/조작됨" 우선
